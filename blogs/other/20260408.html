Aqiu2026/04/08日志日志

开发环境与系统缓存清理全指南

本指南涵盖了主流开发工具（Docker, Python, Conda, Node.js）及系统的缓存清理方案，建议定期执行以释放磁盘空间。

1. 核心开发工具清理

工具清理命令说明
Dockerdocker system prune -a --volumes最彻底：删除所有停止的容器、未使用的网络、悬空镜像及数据卷。
Condaconda clean --all删除索引缓存、锁定文件、未使用过的包和 tarballs。
pnpmpnpm store prune从全局存储中删除当前未被任何项目引用的包。
pippip cache purge删除 pip 下载的所有缓存轮子文件 (.whl) 和源码包。
npmnpm cache clean --force强制清空 npm 本地缓存。

2. Python 项目碎文件清理

在 Python 项目根目录下运行以下命令，可以清除运行时生成的字节码文件：

# 删除所有 __pycache__ 文件夹
find . -type d -name "__pycache__" -exec rm -rf {} +

# 删除所有 .pyc, .pyo 结尾的文件
find . -name "*.py[co]" -delete

3. 前端项目多版本清理 (Node.js)

如果你的电脑里存了大量旧的 JS 项目，每个项目的 node_modules 可能占用数百 MB。

  • 推荐工具：npkill
  • 使用方法：直接在你的代码通用存放目录下运行：
    npx npkill
    使用方向键选择，空格键即可快速删除不再需要的 node_modules。

4. IDE 与系统级清理

VS Code

VS Code 的缓存通常位于以下路径，若遇到插件异常或占用过大，可手动删除：

  • macOS: ~/Library/Application Support/Code/Cache
  • Linux: ~/.config/Code/Cache
  • Windows: %APPDATA%\Code\Cache

JetBrains (PyCharm/IntelliJ)

  • 操作: File -> Invalidate Caches... -> 勾选所有选项 -> Invalidate and Restart。

Homebrew (macOS)

# 清理旧版本的下载包和过期锁
brew cleanup

5. 磁盘占用分析工具

如果清理后空间依然不足，建议使用以下可视化工具定位“大文件”：

  • 命令行 (Linux/macOS): ncdu (推荐，极速分析)
  • Windows: WizTree (扫描速度最快)
  • macOS: DaisyDisk

💡 一键清理脚本 (Shell Script)

你可以将以下内容保存为 clean_dev.sh，赋予权限后定期运行：

#!/bin/bash

echo "开始清理开发环境缓存..."

# 1. Pip
pip cache purge

# 2. Conda (如果安装了)
if command -v conda &> /dev/null; then
    conda clean --all -y
fi

# 3. Docker (需谨慎执行，默认仅清理悬空资源)
docker system prune -f

# 4. pnpm
if command -v pnpm &> /dev/null; then
    pnpm store prune
fi

echo "清理完成！"

注意： 在执行 docker volume prune 或 system prune --volumes 时，请确保你不需要保留任何未运行容器的数据库数据。
