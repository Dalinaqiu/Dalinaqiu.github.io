1 哈希函数

X => H(X)

Y => H(Y)

[思维导图]

  • Collision Resistanse 抗对抗性

Collision是指“哈希碰撞”，即：输入X不等于Y，也可以使得H(X) = H(Y)，不同的输入映射到哈希表同一个位置。

哈希碰撞很难人工创造。

  • Irreversible 单向不可逆性

给定一个X可计算得到H(X)，反过来不行。

  • Puzzle Friendly 易于验证

哈希值的计算是不可预测的，想要H(X)落在某个范围内，只能一个一个去试。

difficult to solve,but easy to verify

常用的hash函数包括：MD5、SHA-1、SHA-2(SHA-256)，其中比特币使用SHA-256算法

import sha256 from 'crypto-js/sha256';

var hash = sha256("Message");

2 签名

区块链用公钥（public key）、私钥（private key）来标识身份，公钥相当于账户ID，转账人通过公钥将比特币转给接受人，公钥是全网可见的。

每一笔交易可以被公钥验证，可以被私钥解密，用来保证信息传递的安全性。

**在比特币区块链中，私钥代表了对比特币的控制权。**交易发起方用私钥对交易（包括转账金额和转账地址）签名并将签名后的交易和公钥广播，各节点接收到交易后可以用公钥验证交易是否合法。在这个过程中交易发起方无需暴露自己的私钥，从而实现保密的目的。

3 比特币的数据结构

3.1 Hash pointers 哈希指针

Block Chain is a linked list using hash pointers.

3.2 Merkle Tree 默克尔树

Merkle Tree是区块链的基本组成部分，Merkle树的叶子结点存储的是数据文件的哈希值，非叶子结点存储的是对其下面所有叶子结点值的组合结果进行哈希计算后得出的哈希值。

一旦某个货币的最新交易已经被足够多的区块覆盖，这之前的支付交易就可以被丢弃以节省磁盘空间。

为便于此而又不破坏区块的哈希值，交易将被哈希进默克尔树 ，只有根节点被纳入到区块的哈希值。老的区块可通过剪除树枝的方式被压缩。树枝内部的哈希不需要被保存。

4 共识协议

PoW (Proof of work)

区块链是由网络上的匿名节点共同维护的，因此比特币要求每个区块都证明其创建过程中投入了大量的工作，以确保想要修改过去区块的不值得信任的节点必须必城市的节点更努力地工作。

5 交易数据

What Is a UTXO?

全称叫：Unspent transaction output open in new window.是为了解决双花问题（Double spending）比特币采用的一种数据结构。

  • transsaction-based ledger 基于交易的模式 （比特币）
  • account-based ledger 基于账户的模式（以太坊、银行等）
